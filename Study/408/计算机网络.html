    一.概论

    # 一.概论

    # 1.什么是Internet？

    网络：包括节点和边，蜘蛛网，大脑的神经元网络，人与人之间的交际网，几十亿台设备的互联网

    计算机网络：是联网的计算机所构成的系统，包括

    • 主机节点（笔记本，手机，iPad，数据服务器，智能冰箱...），它是数据的源，数据的目标.

    • 数据交换节点（路由器，交换机），它是节点，但既不是源也不是数据的目标，它是数据的中转节点，前者工作在网络层，后者工作在链路层，以及一些更高层的负载均衡设备。

    • 链路：把节点连在一起的叫做链路，将主机和主机连在一起，主机和交换节点连在一起，构成一个网络。分为两类：

      • 接入网链路：主机连接到互联网的链路

      • 主干链路：路由器之间的链路

    # 2.什么是协议？

    是支撑互联网工作的一些标准，各个厂商按照协议来制作生产它的网络设备，这些不同的网络设备之间就可以形成互操作，比如虽然我们一台电脑由不同厂商生产的设备组装起来的，他们通过相同的协议就可以共同工作。按照层次的不同，协议可以分成物理层，链路层，网络层（IP），传输层（UDP，TCP），应用层（HTTP，FTP，DNS，WWW，DHCP）协议...

    标准定义：是计算机网络中互相通信的对等实体之间交换信息时所必须遵守的规则的集合。

    协议的特征：

    • 语法
    • 语义
    • 时序

    网络结构：

    • 网络边缘（edge）：
      • 主机
      • 应用程序（客户端和服务器）
    • 网络核心（core）：作用是数据交换
      • 互联着的路由器
      • 网络的网络
    • 接入网，物理媒体（access）：
      • 有线或无线通信链路

    思考下为什么不直接在主机与主机之间点对点的通信呢？

    • 因为如果每次通信都拉根线，类似完全无向图，有2个主机，就有一根线，3个主机就有3根线，4个主机就要5根，依次下去n个主机就需要n * (n - 1)根线，如果n = 10000000呢，那网络规模就非常复杂庞大难以维护.

    边缘（edge）通过接入接入（access）到核心（core），核心把所有的边缘节点接在一起，从而让任意两个端系统之间能够相互通信

    # 3.网络边缘

    网络边缘上运行的应用（淘宝，王者荣耀，抖音，学习通）是网络存在的理由，即需求决定供给。

    • 应用程序以下都算基础设（即应用层以下）
    • 基础设施就是为分布式应用提供服务

    • 面向连接：就是两个应用进程通信之前要握个手打个招呼，两者要做好准备，基础设施好做好准备，为后面的通讯分配好相应的资源，控制变量要做合适的设置。

    • 可靠性：原原本本，不重复不丢失不失序。数据传输的过程中有各种原因会导致数据发生变化，这是我们不愿意看到的，TCP协议就是将下层的不可靠变成可靠

    • 流量控制：水管运输水一样，如果发送方流量大流速快，接收方接口小流速慢就会出现问题，TCP的流量控制就是协调发送方发送速度和接收方的接收速度，从而让两者有序进行

    • 拥塞控制：往往一条链路上有多个主机在使用进行通信，那么就需要判断链路是否通畅，来判断是否可以发送，或者是否可以加快或放慢发生速度等，就像开车一样，对路道的堵塞程度来控制车辆的通行

    • TCP一会要考虑对方的感受一会考虑网络的感受，多愁善感的TCP自然也会内耗（排序，编号，定时重传等等），这些都是更耗时的，在某些场景下难以接受这个代价，比如实时多媒体（直播，视频..），所以UDP和TCP没有绝对的好与坏，只有适不适合场景

    # 5.网络核心：电路交换，分组交换，报文交换

    # 电路交换（Circuit Switching）

    需要建立一条专用的数据通信路径，这条路径上可能包含许多中间节点。这条通信路径在整个通信过程中将被独占，直到通信结束才会释放资源。电路交换适合实时性要求较高的大量数据传输的情况

    优点：

    • 通信时延小。通信双方通过专用线路进行通信，数据可以直达。当数据传输量较大时，优点将十分显著。
    • 线路独占，没有冲突。
    • 实时性强。一旦通信线路建立，双方可以实时通信。

    缺点：

    • 线路独占，利用率太低。
    • 连接建立时间过长。

    计算机的通信有很强的突发性，比如你大部分时间浏览网站（低资源通信），只有在你需要点击下载资源的时候才需要较大的资源通信，所以现在主流基本上都是分组交换，而不是电路交换

    # 分组交换（Packet Switching）

    分组交换采用存储转发技术。实质上是采用了在数据通信的过程中断续（或动态）分配传输带宽的策略（断续分配传输带宽）。

    将报文分成更小的等长的数据段，每一个数据段加上一些控制信息（诸如目的地址和原地址等）后组成首部（header），构成一个分组（packet）。分组又称为“包”，分组的首部也可称为“包头”。

    # 二.应用层

    网络应用是计算机网络存在的理由，如果我们不能构想出任何有用的应用，也就没有任何必要去设计支持它们的网络协议了

    # 1.应用层协议原理

    # 网络应用的体系结构：

    1.客户-服务器模式（C/S：client/server）

    服务器随着请求数的增加，性能不是平滑的下降，是断崖式的下降

    2.对等模式（P2P：Peer To Peer）

    P2P的性能可以是随着客户数量的增加性能平缓下降，很容易能把客户扩容到几百万，这在CS模式是难以想象的。

    但是管理比较困难，每个节点随时可能上下线，不便于管理

    3.混合体：客户-服务器和对等体系结构

    类似QQ聊天，你上线的时候告诉服务器，并注册个IP，然后服务器可以告诉你你的好友IP在哪，然后你们就可以通信了。所以它这目录查询，登记注册这块是CS模式，用户之间对话就是P2P的方式

    # 进程通信

    • 如果两个进程在同一台计算机上，他们之间通信就不需要符合网络的标准，可以借助OS本身提供的进程之间通信方式来通信，比如管道，消息队列，共享缓冲区，信号量等。

    • 当然如果两个进程跑在不同的联网主机上，那么他们之间通信就要符合相应的网络标准，通过交换报文来通信

    # 分布式进程通信需要解决的问题

    1. 进程得有自己的唯一标识（SAP），跟别的进程区分开来，同时这个标识还得具备寻址的功能，它得让别人找得到，所以标识里面要包含一些地址信息。在网络层，采用了用IP地址来区分不同主机，在传输层则采用端口号port来区分同一主机上的不同进程，UDP和TCP都有各自的端口号空间，大小为2^16 ,即0~65535端口号

      任何一对应用进程之间的通信都可以用两个end point 来表示端节点。SAP = IP + UDP/TCP + port

    2. 类似于寄件，你要将SDU（service data unit服务数据单元，应用层是报文）发给别人，就得告诉层间接口你发的包裹是什么（报文），寄件人（发送报文的进程的标识SAP：IP + TCP/UDP 端口），收件人（对方进程的SAP：对方的IP + UDP/TCP端口） 快递公司（传输层实体），要对上述信息进行打包封装寄出.

      即我是谁（报文），我从哪里来（自己IP + 端口），我到哪里去（对方IP + 端口）

      # 套接字（Socket）

      Socket是进程和计算机网络之间的接口，是同一台主机内应用层与运输层之间的接口，不同主机上的两个进程想要通信就必须通过下面的网络。进程通过Socket这样的软件接口向下层网络发送报文或接受报文。类似与一个房子的门户

      套接字是建立在网络应用程序的可编程接口，因此套接字也称为应用程序和网络之间的应用程序编程接口（Application Programming Interface，即API）

      TCP Socket用一个整数来表示四元组（源ip，源端口，目标ip，目标端口），UDP Socket是二元组（因为不需要建立连接，所以只要目标IP和目标端口）

      UDP Socket类似匿名寄件

      3.如何使用传输层提供的服务来实现应用

      # 应用层协议

      协议就是计算机网络互相通信的对等实体之间的交换信息所需要遵守的规则的集合，这里的对等实体是两个端系统的应用层

      现在我们一般访问网站都是用HTTPS协议，它是建立在TCP之上的加密超文本协议，相当于SSL + HTTP，HTTP全称是超文本传输协议，是一种用于在网络上传输数据的协议，通常用于从Web服务器请求并获取网页，图片，视频和其他资源。

      因为TCP/UDP都是明文传输的，如果你的账户密码也这样传输，那明文很容易被他人获取，非法利用造成损失

    # 2.Web和HTTP

    Web是一种应用，HTTP是支持Web的协议

    port一般采用默认端口号，支持匿名访问的可以不提供用户名和口令，HTTP默认80端口

    无状态不需要维护历史信息 ，简单

    # 3.FTP（file Transfer Protocol）

    它是有状态的协议，控制命令和数据传输是在两个TCP连接上进行的（前者21号端口，后者20号），控制连接是客户端主动的，

    数据连接是服务器主动进行的

    Retrieve filename从服务器下载文件到客户端，Store filename从客户端上载文件到服务器

    # 4.Email

    • 用户代理：Web应用的用户代理是浏览器，FTP的用户代理是FTP客户端软件，主机是通过客户端软件跟服务器打交道 ，这个软件就是应用的用户代理，它代理你去执行一些动作
    • 邮件服务器：用户代理配置好邮件服务器的ip和端口号，将邮件发给邮件服务器，邮件服务器将邮件放入队列中，挨个输出队列中的邮件到目标的邮件服务器（邮件服务器之间用SMTP协议通信），目标的邮件服务器发送到相应用户的邮箱中。
      • 收取邮件：目标用户运行客户代理再从邮箱中，把别人发送给他的邮件再拉过来（拉的协议就有POP3，IMAP，HTTP）

    所以也限制了邮件传输内容必须在7位ASCII码范围之内，所以传不了中文，也传不了附件之类的，后续补丁解决了这个问题
